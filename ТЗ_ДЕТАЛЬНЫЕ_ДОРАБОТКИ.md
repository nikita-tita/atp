# Подробный перечень доработок платформы торговли авиатехникой

## 1. ЛОКАЛИЗАЦИЯ И КОНТЕНТ

### 1.1 Перевод интерфейса
**Приоритет: Критический**
- Полный перевод всех текстов с русского на английский
- Адаптация терминологии под авиационную индустрию:
  - "джеты" → "aircraft/airliners"
  - "самолеты" → "aircraft"
  - "лизинг" → "leasing" (dry lease/wet lease)
- Создание глоссария авиационных терминов
- Проверка переводов носителем языка из авиационной сферы

### 1.2 Замена визуального контента
- Замена всех изображений на реальные фотографии коммерческих самолетов
- Создание библиотеки изображений по моделям:
  - Boeing (737, 747, 777, 787)
  - Airbus (A320, A330, A340, A350, A380)
  - Embraer (E-Jets серии)
  - Bombardier и другие
- Стандартизация размеров и качества изображений
- Добавление технических схем самолетов

## 2. СТРУКТУРА КАТАЛОГА И НАВИГАЦИЯ

### 2.1 Основные разделы каталога
Три главных категории:

#### 2.1.1 Продажа самолетов (Aircraft Sales)
- **Предложения (Listings)**: Самолеты, выставленные на продажу
- **Запросы (Requests)**: Заявки покупателей на поиск определенных моделей

#### 2.1.2 Лизинг (Aircraft Leasing)
- **Dry Lease**: Аренда самолета без экипажа и обслуживания
- **Wet Lease**: Аренда с экипажем и частичным обслуживанием
- Отдельные формы для каждого типа лизинга

#### 2.1.3 ACMI (Aircraft, Crew, Maintenance, Insurance)
- Полный комплекс услуг: самолет + экипаж + обслуживание + страховка
- Специальные поля для указания:
  - Срок аренды
  - География полетов
  - Требования к экипажу
  - Условия обслуживания

### 2.2 Навигация и фильтры
Единая система фильтров для всех разделов:
- Производитель (Boeing, Airbus, Embraer и др.)
- Модель самолета
- Год выпуска (диапазон)
- Цена (диапазон)
- Статус (новый/подержанный)
- Местоположение
- Часы налета (диапазон)
- Количество циклов

## 3. СИСТЕМА РЕГИСТРАЦИИ И ВЕРИФИКАЦИИ

### 3.1 Многоуровневая регистрация

#### 3.1.1 Базовая регистрация
Обязательные поля:
- ФИО контактного лица
- Email (с обязательной верификацией)
- Телефон (с SMS верификацией)
- Название компании
- Должность в компании
- Страна регистрации компании
- Тип деятельности (брокер/авиакомпания/лизинговая компания/частное лицо)

#### 3.1.2 Документальная верификация (KYC)
По образцу платформы Aviasi:
**Загрузка документов компании:**
- Регистрационные документы
- Лицензии (авиационные, брокерские)
- Налоговые документы

**Верификация личности:**
- Паспорт/ID руководителя
- Доверенность (если регистрируется не руководитель)
- Подтверждение адреса компании
- Финансовые документы (банковские справки)

#### 3.1.3 Дополнительная верификация для доступа к конфиденциальным данным
- Подписание NDA (соглашения о неразглашении)
- Загрузка мандата от клиента (для брокеров)
- Подтверждение финансовой состоятельности

### 3.2 Статусы пользователей
- **Гость** - просмотр базовой информации
- **Зарегистрированный** - доступ к расширенной информации
- **Верифицированный** - доступ к техническим характеристикам
- **С мандатом** - полный доступ ко всей информации

## 4. ЛОГИКА ДОСТУПА К ИНФОРМАЦИИ

### 4.1 Уровни видимости данных

#### 4.1.1 Публичная информация (для всех)

- Год выпуска

- Количество циклов
- Основные характеристики

#### 4.1.3 Конфиденциальная информация (верифицированные с мандатом)

## 5. ДЕТАЛЬНЫЕ ХАРАКТЕРИСТИКИ САМОЛЕТОВ

### 5.1 Обязательные технические поля

#### 5.1.1 Основная информация
- **Производитель**: выпадающий список (Boeing, Airbus, Embraer, Bombardier, др.)
- **Модель**: зависимый выпадающий список
- **Серийный номер (MSN)**: текстовое поле (скрытое до верификации)
- **Регистрационный номер**: текстовое поле (скрытое до верификации)
- **Год выпуска**: числовое поле
- **Дата первого полета**: календарь

#### 5.1.2 Техническое состояние
- **Общие часы налета**: числовое поле
- **Часы по двигателям**: отдельные поля для каждого двигателя
- **Общие циклы**: числовое поле
- **Циклы по двигателям**: отдельные поля
- **Количество посадок**: числовое поле

#### 5.1.3 Конфигурация салона
- **Общее количество мест**: числовое поле
- **Бизнес-класс**: количество мест
- **Премиум-эконом**: количество мест
- **Эконом-класс**: количество мест
- **Тип конфигурации**: выпадающий список

#### 5.1.4 История обслуживания (Чеки)
Система чеков A, B, C, D с интервалами:
- **A-Check** (каждые 500-800 часов)
  - Дата последнего выполнения
  - Дата следующего (автоматический расчет)
  - Место выполнения
  - Статус соответствия
- **B-Check** (каждые 6 месяцев)
- **C-Check** (каждые 18-24 месяца)
- **D-Check** (каждые 6-12 лет) - самый дорогой и важный
- **Специальные директивы AD**: список выполненных
- **Модификации**: список установленного дополнительного оборудования

#### 5.1.5 Местоположение и статус
**Текущее местоположение:**
- Страна (выпадающий список)
- Город (зависимый список)
- Аэропорт (код IATA/ICAO)

**Статус использования:**
- В эксплуатации
- На хранении
- На обслуживании
- Выведен из эксплуатации
- **Готовность к передаче**: календарь

### 5.2 Дополнительное оборудование
- **Авионика**: список установленного оборудования
- **Интерьер**: описание состояния и особенностей
- **Двигатели**: модель, часы, циклы по каждому
- **APU** (Auxiliary Power Unit): часы, циклы, состояние
- **Шасси**: циклы, последний ремонт

### 5.3 Документация
- **Технический паспорт**: файл PDF
- **Сертификат летной годности**: срок действия
- **Страховка**: покрытие, срок действия
- **Логбуки**: сканы основных страниц
- **История ремонтов**: подробные записи

## 6. ЛИЧНЫЙ КАБИНЕТ

### 6.1 Дашборд (главная страница кабинета)

#### 6.1.1 Виджеты статистики
- **Активные объявления**: количество с разбивкой по типам
- **Просмотры за период**: график по дням/неделям/месяцам
- **Входящие сообщения**: количество непрочитанных
- **Активные запросы на compliance**: входящие и исходящие
- **Избранное**: количество сохраненных объявлений
- **Статистика конверсий**: процент откликов на объявления

#### 6.1.2 Последняя активность
- Лента последних действий на платформе
- Уведомления о новых сообщениях
- Статус проверки загруженных документов
- Напоминания о продлении подписки

### 6.2 Управление объявлениями

#### 6.2.1 Список объявлений
Таблица с колонками:
- Модель самолета
- Цена
- Статус (активно/на модерации/снято)
- Количество просмотров
- Дата создания/обновления
- Действия (редактировать/удалить/продлить)

#### 6.2.2 Создание/редактирование объявления
Пошаговая форма:
- **Шаг 1**: Основная информация
- **Шаг 2**: Технические характеристики
- **Шаг 3**: Загрузка фотографий
- **Шаг 4**: Настройки приватности
- **Шаг 5**: Проверка и публикация
- Автосохранение черновиков
- Предпросмотр объявления перед публикацией

#### 6.2.3 Аналитика по объявлениям
- График просмотров по дням
- География просмотров
- Источники трафика
- Действия пользователей (сохранение в избранное, клики на контакты)
- Сравнение с похожими объявлениями

### 6.3 Система сообщений

#### 6.3.1 Внутренняя переписка
**Инбокс с фильтрами:**
- Все сообщения
- Непрочитанные
- Важные (помеченные)
- Архив
- Поиск по переписке
- Групповые чаты для сложных сделок
- Уведомления на email и SMS

#### 6.3.2 Интеграция с запросами
- Автоматическое создание чата при отклике на объявление
- Передача контекста (модель самолета, цена и т.д.)
- Статусы переговоров (активно/завершено/приостановлено)

### 6.4 Управление запросами на раскрытие данных

#### 6.4.1 Входящие запросы (для продавцов)
Список заявок с информацией:
- ФИО и компания заявителя
- Статус верификации заявителя
- Модель запрашиваемого самолета
- Дата запроса
- Приложенный мандат (ссылка на скачивание)

**Действия:**
- Одобрить (раскрыть данные)
- Отклонить (с указанием причины)
- Запросить дополнительную информацию

#### 6.4.2 Исходящие запросы (для покупателей)
Список поданных заявок:
- Модель интересующего самолета
- Статус заявки (на рассмотрении/одобрено/отклонено)
- Дата подачи
- Комментарии от продавца
- Загрузка мандатов для новых запросов
- Отслеживание статуса проверки документов

## 7. WORKFLOW ДЛЯ ЗАЯВОК И РАСКРЫТИЯ ДАННЫХ

### 7.1 Процесс для покупателя/брокера

#### 7.1.1 Первичный интерес
- Просмотр публичной информации о самолете
- Клик "Связаться с продавцом"
- Проверка статуса регистрации:
  - Если не зарегистрирован → редирект на регистрацию
  - Если зарегистрирован → переход к следующему шагу

#### 7.1.2 Подача запроса
**Выбор типа запроса:**
- Базовый (только контакты)
- Расширенный (технические данные)

**Заполнение формы:**
- Информация о клиенте
- Серьезность намерений
- Временные рамки сделки

**Загрузка мандата (для расширенного запроса):**
- Letter of Intent (LOI)
- Proof of Funds
- Broker Agreement

#### 7.1.3 Ожидание одобрения
- Автоматическое уведомление о получении запроса
- Проверка мандата администрацией платформы (1-2 рабочих дня)
- Передача на рассмотрение продавцу
- Получение финального решения

### 7.2 Процесс для продавца

#### 7.2.1 Настройка объявления
**Выбор уровня безопасности:**
- Открытый (контакты доступны сразу)
- Умеренный (требуется регистрация)
- Закрытый (требуется мандат)
- Настройка автоответов для типовых вопросов
- Делегирование обработки запросов помощникам

#### 7.2.2 Обработка входящих запросов
- Получение уведомления о новом запросе
- **Просмотр профиля заявителя:**
  - Статус верификации
  - История сделок на платформе
  - Рейтинг от других продавцов
- **Анализ мандата:**
  - Финансовые возможности
  - Серьезность намерений
  - Соответствие требованиям
- **Принятие решения:**
  - Одобрить полностью
  - Одобрить частично (ограниченные данные)
  - Отклонить с указанием причин
  - Запросить дополнительную информацию

### 7.3 Автоматизация процессов

#### 7.3.1 Умные уведомления
- Мгновенные уведомления о критически важных событиях
- Дайджесты по активности за день/неделю
- Напоминания о непринятых решениях
- Эскалация для просроченных запросов

#### 7.3.2 Интеграция с внешними системами
- Проверка финансовых документов через банковские API
- Верификация авиационных лицензий через государственные базы
- Автоматическая проверка санкционных списков

#### 8.1.1 Очередь на верификацию
- Список новых регистраций с приоритизацией
- Чеклист проверки для каждого типа пользователей
- Интеграция с внешними базами для автоматической проверки
- Система оценки рисков для каждого заявителя

#### 8.1.2 Управление документами
- Просмотрщик документов с возможностью комментирования
- Шаблоны отказов с типовыми причинами
- История изменений статусов пользователей
- Архив всех загруженных документов

#### 8.1.3 Мониторинг активности
- Подозрительная активность: множественные аккаунты, поддельные данные
- Жалобы пользователей на некачественные объявления или мошенничество
- Автоматические флаги для проблемных паттернов поведения

### 8.2 Управление контентом

#### 8.2.1 Модерация объявлений
- Очередь на проверку новых объявлений
- Автоматическая проверка на дубликаты и фейки
- Верификация технических данных против баз данных производителей

- Анализ рыночных цен по моделям самолетов
- Флаги для подозрительно низких/высоких цен
- Исторические тренды ценообразования

### 8.3 Аналитика и отчетность

#### 8.3.1 Операционные метрики
- Количество регистраций по периодам
- Конверсия регистраций в верифицированных пользователей
- Среднее время обработки заявок
- Количество успешных сделок
- География пользователей

#### 8.3.2 Финансовая отчетность
- Доходы от подписок по тарифным планам
- Комиссии с успешных сделок
- Costs per acquisition новых пользователей
- LTV (lifetime value) клиентов
- Прогнозы доходности

## 10. ТЕХНИЧЕСКАЯ ИНФРАСТРУКТУРА

### 10.1 Безопасность данных

#### 10.1.1 Шифрование и хранение
- End-to-end шифрование для конфиденциальных данных
- Раздельное хранение публичной и приватной информации
- Регулярные бэкапы с тестированием восстановления
- Audit log всех действий с чувствительными данными

#### 10.1.2 Контроль доступа
- Ролевая модель с гранулярными правами
- Двухфакторная аутентификация для всех пользователей
- Session management с автоматическим завершением
- IP whitelisting для особо важных аккаунтов

#### 10.1.3 Соответствие требованиям
- GDPR compliance для европейских пользователей
- Соглашения о неразглашении в цифровом формате
- Audit trail всех изменений данных
- Right to be forgotten - возможность полного удаления данных

### 10.2 Email и SMS системы

#### 10.2.1 Верификация контактов
- Email верификация при регистрации с защитой от спама
- SMS верификация телефонов с международной поддержкой
- Проверка на одноразовые email адреса и виртуальные номера
- Blacklist для проблемных доменов и номеров

#### 10.2.2 Уведомления и рассылки
- Transactional emails: подтверждения, статусы заявок
- Marketing campaigns: новости, обновления продукта
- Critical alerts: проблемы с безопасностью, срочные уведомления
- Персонализация контента под профиль пользователя

### 10.3 Производительность и масштабируемость

#### 10.3.1 Архитектура
- Микросервисная архитектура для независимого масштабирования
- CDN для быстрой загрузки изображений глобально
- Database sharding для больших объемов данных
- Caching strategy с Redis/Memcached

#### 10.3.2 Мониторинг
- Application Performance Monitoring (APM)
- Real-time alerting для критических метрик
- User experience monitoring для выявления проблем UX
- Capacity planning на основе прогнозов роста

## 11. МОБИЛЬНАЯ АДАПТАЦИЯ
